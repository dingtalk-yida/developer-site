待办任务表格展示

案例背景​

当系统自带我的待办页面满足不了个性化的需求时，需要用到自定义页面展示待办的内容，可以参考以下案例来满足大家的需求。

实现效果​

执行以下操作，实现的效果，如下：

  1. 展示流程中未处理数据
  2. 展示自定义页面获取到的数据
  3. 在流程表单添加数据
  4. 查看待办数据是否新增到了自定义页面

操作步骤​

步骤 1：创建自定义页面​

  1. 路径：进入已创建好的应用 >> 页面左上角+号 >> 新建自定义页面

  2. 拉取自定义页面表格组件

  3. 添加 5 个字段

标题数据字段数据类型
processInstanceIdprocessInstanceId文本
originatorNameoriginatorName文本
titletitle文本
originatorPhotooriginatorPhoto文本
createTimecreateTime时间

  1. 表格组件添加字段

  2. 修改字段数据类型

步骤 2：创建数据源及配置​

  1. 创建数据源 名称：getData 请求地址：/dingtalk/web/应用唯一编码/v1/task/getTodoTasksInApp.json 请求方式：GET 请求参数： pageSize：100 currentPage：1 数据源配置

  2. 数据源添加 didFetch 函数 didFetch 函数为请求完成回调函数，主要用于数据返回后的数据处理 添加 didFetch 函数 代码示例： 将返回的数组处理为表格的格式后绑定数据源

const value = [];
const data = content.data.map((item) => {
  let arr = {
    processInstanceId: item.processInstanceId,
    originatorName: item.originatorName,
    title: item.title,
    originatorPhoto: item.originatorPhoto,
    createTime: item.createTime,
    instid: item.processInstanceId,
  };
  value.push(arr);
  console.log(arr);
});
console.log(value);
let result = {
  data: value,
  currentPage: content.currentPage,
  totalCount: content.totalCount,
};
console.log(result);
return result;
